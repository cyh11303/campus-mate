# Campus Mate DB Persistent Version

이 버전은 기존 메모리 기반 저장 방식을 SQLite 데이터베이스 기반 영속 저장 방식으로 확장한 버전입니다.

## 실행 방법

```bash
pip install -r requirements.txt
uvicorn server:app --reload
```

접속 주소:

```text
http://127.0.0.1:8000
```

실행하면 프로젝트 폴더에 `campus_mate.db`가 자동 생성됩니다.

## DB 구성

- `users`: 룸메이트 등록 정보
- `meals`: 밥약 신청 정보
- `gallery_posts`: 음식 갤러리 게시글 및 리액션
- `trait_weights`: 생활 성향별 가중치

## 2026-06-01 최신 변경사항

### 1. 본인 흡연 여부 추가

룸메이트 등록 화면의 기본 정보 영역에 `본인 흡연 여부` 선택 칸을 추가했습니다.

DB의 `users` 테이블에는 다음 컬럼이 추가됩니다.

```sql
smoke TEXT DEFAULT ''
```

기존 DB를 그대로 사용하는 경우에도 서버 실행 시 자동으로 `smoke` 컬럼을 추가하도록 마이그레이션 코드를 넣었습니다.

### 2. 흡연 여부 절대 필터링

매칭 알고리즘에서 흡연 여부는 절대 조건으로 처리됩니다.

- 흡연자는 흡연자끼리만 매칭
- 비흡연자는 비흡연자끼리만 매칭

또한 사용자가 `선호 룸메이트 흡연 여부`를 지정한 경우 해당 조건도 함께 확인합니다.

### 3. 아침형 인간 / 야행성 성향 제거

평균 기상 시간과 평균 취침 시간 입력값과 의미가 겹치므로, 생활 성향 체크리스트에서 아래 항목을 제거했습니다.

- 아침형 인간
- 야행성

기상/취침 시간은 별도의 시간대 유사도 보정 요소로 계속 사용됩니다.

### 4. 가중치 최대값 2.0으로 제한

생활 성향 가중치 범위를 다음과 같이 조정했습니다.

```text
기존: 1.0 ~ 3.0
변경: 1.0 ~ 2.0
```

프론트엔드 슬라이더, 직접 입력값, 백엔드 저장 로직 모두 2.0을 초과하지 않도록 수정했습니다.
기존 DB에 2.0보다 큰 값이 저장되어 있어도 서버 실행 시 2.0으로 보정됩니다.

## 배포 주의사항

SQLite 파일과 `uploads/` 폴더는 로컬에서는 유지되지만, Render/Railway 같은 배포 환경에서는 재배포 시 사라질 수 있습니다.
최종 배포에서는 Persistent Disk/Volume 또는 PostgreSQL, S3/Cloudinary 같은 외부 저장소 사용을 권장합니다.

## v3 변경 사항

### 룸메이트 매칭
- 기본 정보에 본인 흡연 여부를 유지합니다.
- `선호 룸메이트 성별`, `선호 룸메이트 흡연 여부` 선택 항목은 제거했습니다.
- 백엔드의 Primary Filter에서 성별과 흡연 여부를 절대 조건으로 강제합니다.
  - 남자는 남자끼리, 여자는 여자끼리만 매칭됩니다.
  - 흡연자는 흡연자끼리, 비흡연자는 비흡연자끼리만 매칭됩니다.
- 생활 성향 체크리스트에서 `아침형 인간`, `야행성`은 제거했고, 평균 기상/취침 시간으로 생활 패턴을 반영합니다.
- 성향 가중치 범위는 1.0~2.0으로 제한됩니다.

### 밥약 시스템
- 밥약 신청 시 연락수단을 함께 입력합니다.
- 연락수단은 신청자 목록 카드에 바로 표시하지 않고, 밥약 요청이 수락된 뒤에만 요청함에서 확인할 수 있습니다.
- 신청자 카드에 `같이 먹기 요청` 버튼을 추가했습니다.
- `밥약 요청함`에서 내 학번을 입력하면 받은 요청과 보낸 요청을 확인할 수 있습니다.
- 받은 요청은 수락/거절할 수 있으며, 수락 시 상대방의 연락수단이 공개됩니다.

### 추가/변경된 DB 구조
- `meals.contact`: 밥약 신청자의 연락수단 저장
- `meal_requests`: 밥약 요청 저장
  - `id`
  - `target_student_id`
  - `requester_student_id`
  - `requester_name`
  - `requester_contact`
  - `message`
  - `status`: `pending`, `accepted`, `rejected`
  - `created_at`

## v4 수정 사항
- 밥약 요청을 보낸 직후 요청함 학번 입력칸을 자동으로 채우거나 보낸 요청을 자동 조회하지 않도록 수정했습니다.
- 요청함은 사용자가 직접 10자리 학번을 입력하고 `요청함 확인` 버튼을 눌렀을 때만 조회됩니다.
- 입력한 학번이 요청받은 사람의 학번이면 `받은 요청`, 요청을 보낸 사람의 학번이면 `보낸 요청`에 각각 표시됩니다.
